# LiveHub 美食分享平台 —— 分布式本地生活服务系统

## 1. 项目简介

LiveHub 是一个基于 Spring Cloud Alibaba 深度定制的分布式本地生活服务系统（仿大众点评）。项目通过微服务架构实现了用户认证、商户管理、优惠券秒杀、社交 Feed 流及数字化签到等核心业务。

项目重点攻克了高并发下的数据一致性、微服务链路下的身份透传以及多级缓存的高性能同步等技术难点。

## 4. 核心亮点与难点突破

### 4.1 鉴权体系：网关魔改与身份透传方案

针对微服务链路下用户信息丢失和 Redis 重复查询的痛点，设计并实施了**“一次鉴权，全链路信任”**方案：
- **网关拦截**：基于 SaReactorFilter 实现响应式近源拦截，未登录请求直接阻断。
- **Header 编码透传**：通过自定义 GlobalFilter 利用 mutate() 机制将用户信息 JSON 化并进行 **URL 编码**，注入自定义请求头。
- **自动装配注入**：在 **Common 模块** 封装拦截器，通过 Spring Boot 3 的 AutoConfiguration 自动为各微服务注入用户信息至 ThreadLocal，实现业务层无感获取 UserHolder。


### 4.2 秒杀系统：高可靠异步下单链路

为了支撑高并发秒杀场景，构建了 **“Redis 预扣 + MQ 削峰”** 的闭环逻辑：
- **原子化校验**：编写 **Lua 脚本** 集成“库存检查、一人一单校验、预扣库存”三步逻辑，确保不超卖、不重下。
- **MQ 可靠性闭环**：
  - **发送侧**：开启 **Publisher Confirm** 模式，确保消息安全送达交换机。
  - **消费侧**：配置 **Manual ACK**（手动应答），仅在订单落库成功后确认，结合**指数退避重试**机制。
  - **兜底侧**：设置 **死信队列 (DLQ)** 监控积压和处理异常，通过运维接口实现 Redis 与 DB 的数据对账修复。


### 4.3 高性能缓存：Caffeine + Redis 二级缓存
为压榨商户详情接口性能，设计了基于 **Caffeine**（本地内存）与 **Redis**（分布式）的二级缓存体系：
- **读取策略**：遵循 **L1 (Caffeine) -> L2 (Redis) -> DB** 路径，大幅减少网络 I/O。
- **一致性同步**：利用 RabbitMQ 的 **Fanout 广播模式** 配合**临时匿名队列**。当商户信息变更时，广播失效指令至所有服务实例，同步清理 JVM 内存，保证数据一致性。
- **健壮性防护**：采用 **Redisson 互斥锁** 解决缓存击穿，通过空值缓存策略防止恶意穿透。

### 4.4 社交 Feed 流与数据统计

- **滚动分页**：利用 Redis **Sorted Set** 存储 Feed 流，采用时间戳作为 Score，实现基于 reverseRangeByScore 的滚动分页，解决传统分页导致的动态列表数据重漏问题。
- **精简统计**：利用 **Redis Bitmap** 实现亿级用户签到数据存储，配合位运算实现连续签到统计，空间利用率提升百倍。

## 5. 项目结构

```

├── common            # 公共模块：DTO、工具类、拦截器、全局配置

├── shop-service      # 商户服务：商户CRUD、二级缓存逻辑、GEO搜索
├── voucher-service   # 优惠券服务：券管理、库存管理
├── order-service     # 订单服务：秒杀逻辑、RabbitMQ消费端、分布式事务
└── social-service    # 社交服务：探店笔记、点赞排行榜、Feed流推送
```

## 6. 快速启动
### 6.1 基础设施（Docker 一键启动）
`cd config`
`docker-compose up -d`
包含：Nacos(8848), RabbitMQ(5672), Redis(6379), Seata(8091)

### 6.2 数据库初始化
1. 创建 MySQL 库 hm_dianping。
2. 执行 docs/SQL/ 目录下所有脚本（包含业务表、undo_log、以及最新的 tb_shop 结构）。

### 6.3 启动顺序
1. 编译公共依赖：mvn clean install -DskipTests（在根目录执行）。
2. 按顺序启动：**User** -> **Shop** -> **Voucher** -> **Order** -> **Social** -> **Gateway**。

## 7. 结语
本项目不仅是一个功能完整的业务系统，更是对微服务分布式治理的一次深度实践。通过对 **Sa-Token、RabbitMQ、Seata** 等组件的深度整合，实现了高性能与高可靠性的平衡。
